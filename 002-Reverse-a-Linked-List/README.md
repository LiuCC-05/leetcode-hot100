# 002. 反向链表
### 📝 题目描述
给你单链表的头节点 head ，请你反转链表，并返回反转后的链表。

### 🔑 关键知识
1. **定义参数**：pre，curr，nxt;
2. **初始化**:pre =None,curr=head;
   
### 🛠️ 避坑小结
1. **顺序不能乱**：先存 `nxt`，再改 `curr.next`。
2. **返回值是 prev**：当 `curr` 跑过头变成 `None` 时，`prev` 刚好站在新链表的最高点。
3. **断后很重要**：第一个节点的 `.next` 必须指向 `None`，否则链表会变成一个无限旋转的环！
   
### 🧠 深刻教训：关于“旋转环”的领悟

*   **现象**：不设置 `prev = None`，链表会变成 `1 <-> 2` 互指。
*   **本质**：链表节点的 `.next` 属性是“执着”的。如果不手动将其指向 `None` 或新节点，它会保留旧地址。
*   **后果**：遍历时陷入 `1->2->1->2` 的无限死循环，导致 TLE (Time Limit Exceeded)。
*   **结论**：反转的第一步，必须是给原本的“头节点”找一个 `None` 做归宿。
